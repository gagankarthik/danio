◆

Tiny & dependency-free

The whole engine — VDOM, reconciler, hooks, store, router — gzips to ~8 kB and pulls in nothing else.

★

Instantly familiar

Components, JSX, useState, useEffect, context, a Redux-style store. Nothing new to learn.

❤

Actually readable

Every file is plain, commented JavaScript. Open node_modules/danio/src — the framework is right there.

⚡

Fast by default

Bailouts mean a setState re-renders one component, not the tree — ~200× faster updates on a 500-row bench.

⌘

Batteries included

Store with middleware, a History-API router, error boundaries, memo, and SSR — all in the box.

◧

SSR + hydration

Render to HTML on the server for SEO and first paint, then hydrate the exact same nodes on the client.

Write it the way you already think.

  • ✓A component is just a function that returns markup.
  • ✓useState gives it memory; the setter re-renders it.
  • ✓No providers, no config, no boilerplate to boot up.
  • ✓Ships with TypeScript types and works with Tailwind or any CSS.

Counter.jsx
import { render, useState } from 'danio'

function Counter() {
  const [n, setN] = useState(0)
  return (
    <button onClick={() => setN(n + 1)}>
      clicked {n} times
    </button>
  )
}

render(<Counter />, document.getElementById('root'))
// that's the whole app.
